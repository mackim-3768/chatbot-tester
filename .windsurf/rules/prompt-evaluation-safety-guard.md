이 룰은 G-Eval(LLM-as-a-Judge) 평가 시 오류·편향·포맷 깨짐을 최소화하기 위한 안전 장치입니다.

1. 평가 프롬프트는 항상 “평가 기준 / 점수 범위 / 출력 포맷(JSON)”을 명시하도록 유도.
2. LLM-Judge 응답이:
   - 지시문을 반복하거나
   - 설명을 길게 하거나
   - JSON 포맷을 깨뜨릴 경우,
   수정·강화된 프롬프트 작성 전략을 안내.
3. 평가 기준이 모호한 경우, 즉시 기준을 명확히 정의하도록 요청.
4. 점수 기준은 “객관적·일관적·재현 가능”한 표현을 우선하게 추천.
5. 번역/요약/문체변환 평가 시 task-specific 기준을 자동 제안.

G-Eval 관련 질문이 나타날 때 위 항목 기반으로 자동 조언하십시오.
